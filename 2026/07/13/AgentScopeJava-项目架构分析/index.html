AgentScopeJava 项目架构分析

Posted on 2026-07-13 In AgentScopeJava Views:

1. 概览

本文是对开源项目 agentscope-java 进行的项目架构分析，以便帮助对该项目进行结构化理解与记忆。

  • 项目开源地址：https://github.com/agentscope-ai/agentscope-java
  • 项目官网地址：https://java.agentscope.io/v2/zh/docs/index.html

3. 整体分层架构

4. 模块职责速览

模块 定位 核心能力
agentscope-core 框架基石 Agent 模型、LLM 集成 (OpenAI/DashScope/Gemini/Anthropic/Ollama)、Tool 系统、MCP 协议、消息管道、RAG 框架、规划系统、技能系统、会话管理、链路追踪
agentscope-harness 增强运行时 沙箱执行 (Docker/K8s/E2B/Daytona)、工作区管理、子 Agent 编排、文件系统管理、内存压缩、快照 (Redis/OSS)
agentscope-extensions 可选扩展集 A2A 协议、AG-UI、多种记忆后端、多种 RAG 后端、调度器、会话存储、技能仓库、服务发现、消息队列、训练数据、Kotlin 协程桥接
agentscope-distribution 分发打包 agentscope-all (shaded uber-JAR) + agentscope-bom (版本 BOM)
agentscope-examples 示例项目 多种 Agent 模式示例、框架集成示例

5. 核心依赖关系图

关键设计原则：

  • core 是唯一的基础：所有模块最终都依赖 agentscope-core，core 本身零内部依赖
  • 扩展模块声明为 optional：大部分扩展使用 <optional>true</optional> + <scope>provided</scope>，消费者按需引入
  • 技术栈：Java 17 + Reactor (响应式) + Jackson + OkHttp + SLF4J

8. 架构设计要点总结

  1. Core 是纯粹的核心抽象：Agent 模型、LLM 集成、工具体系、消息管道，零内部依赖，是整个框架唯一的基础
  2. Harness 补充运行态能力：构建在 Core 之上的增强层，提供沙箱执行、工作区管理、子 Agent 编排等生产级能力
  3. Extensions 插件化按需扩展：27 个子模块覆盖记忆、RAG、调度、会话、技能仓库、服务发现、消息队列等，全部通过 optional + provided 机制由消费者按需引入
  4. agentscope-all 提供一站式依赖：Shaded uber-JAR 让不想管理多模块依赖的用户只需引入一个 artifact
  5. 多框架集成：Spring Boot、Micronaut、Quarkus 三大 DI 框架的自动配置 Starters
  6. 多 LLM 厂商支持：OpenAI、DashScope (阿里)、Gemini (Google)、Anthropic (Claude)、Ollama
  7. A2A 协议是核心集成模式：Google Agent-to-Agent 协议深度集成（客户端/服务端/RocketMQ 传输/Nacos 服务发现/Spring Boot 自动配置）
  8. 阿里云生态：DashScope、百炼、Nacos、OSS、RocketMQ 等阿里云服务有多项集成
