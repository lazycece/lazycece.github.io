AgentScopeJava 项目架构分析

Posted on 2026-07-13 In agentscope-java Views:

1. 概览

本文是对开源项目 agentscope-java（v2.0版本）进行的项目架构分析，以便帮助对该项目进行结构化理解与记忆。

  • 项目开源地址：https://github.com/agentscope-ai/agentscope-java
  • 项目官网地址：https://java.agentscope.io/v2/zh/docs/index.html

2. 顶层目录结构

3. 整体分层架构

关键设计原则：

  • core 是唯一的基础：所有模块最终依赖 agentscope-core，core 本身零内部依赖
  • 扩展模块分组管理：扩展按功能域分组（model / rag / mem / channel / sandbox / scheduler / nacos / skills / protocol），每个分组是一个父 POM
  • 扩展声明为 optional：大部分扩展使用 <optional>true</optional> + <scope>provided</scope>，消费者按需引入
  • 技术栈：Java 17 + Reactor (响应式) + Jackson + OkHttp + SLF4J

4. 模块职责速览

模块 定位 核心能力
agentscope-core 框架基石 Agent 模型 (Agent/AgentBase/ReActAgent)、消息系统 (Msg/ContentBlock)、中间件链 (MiddlewareBase)、工具体系 (@Tool/Toolkit)、MCP 协议、权限引擎、RAG 框架、技能系统、AgentState 状态管理、细粒度事件流、链路追踪、凭证管理、优雅停机
agentscope-harness 增强运行时 HarnessAgent（生产级 Agent 运行时）、工作区管理 (WorkspaceManager)、沙箱执行 (Docker)、子 Agent 编排 (SubagentDeclaration)、文件系统 (Local/Remote/Sandbox Overlay)、技能策划 (SkillCurator/Promoter/Gate)、内存压缩 (ConversationCompactor)、中间件栈（14 个内置中间件）、Plan Mode 管理
agentscope-extensions 可选扩展集 模型供应商 (5个)、RAG 后端 (5个)、记忆后端 (3个)、通道集成 (6个)、沙箱后端 (4个)、调度器 (2个)、状态存储 (5个)、技能仓库 (3个)、Nacos 集成 (3个)、协议 (A2A/AG-UI/Chat-Web)、Spring Boot Starters (10个)、Micronaut、Quarkus
agentscope-distribution 分发打包 agentscope (shaded uber-JAR) + agentscope-bom (版本 BOM)
agentscope-examples 示例项目 Agent 示例 (Builder/Paw/Coding/Data/Claw/BobaTea)、集成示例 (A2A/AG-UI/Micronaut/Quarkus)、多Agent模式 (Handoff/Pipeline/Routing/Skills/Subagent/Supervisor/Workflow)

包结构 (io.agentscope.core):

顶层类 (io.agentscope.core):

  • ReActAgent — 主 Agent 实现，执行 ReAct 循环（Reasoning → Acting → Summary）
  • Version — 版本信息

v2 核心架构要点

Agent 接口层次:

ReActAgent 执行循环 (每次迭代: Reasoning → Acting → Summary):

  1. Reasoning: 通过 onReasoning 中间件链 → 调用模型（流式）→ 收集文本/思考/工具调用
  2. Acting: 通过 onActing 中间件链 → 执行工具（并行/串行）→ PermissionEngine 权限检查
  3. Summary: 构建 AssistantMessage（含工具结果），决定是否继续迭代

循环终止条件：模型停止 / 达 maxIters / 中断 / stopOnReject

中间件系统 (v2 替代旧 Hook 体系):

细粒度事件系统 (v2，34 种事件类型):

状态管理 (v2):

  • AgentState 是核心的运行时状态容器（per-session，per-user 隔离）
  • 上下文存储在 AgentState.getContext() (List<Msg>)，而非独立的 Memory 对象
  • AgentStateStore 负责持久化（InMemory / JsonFile）
  • 旧 Memory 接口标记为 @Deprecated

权限系统 (v2 新增):

  • PermissionEngine — 工具调用权限评估
  • PermissionMode: ALWAYS_ALLOW / EXPLORE / ACCEPT_EDITS / HITL_ONLY / CUSTOM
  • PermissionRule — 允许/拒绝模式匹配
  • HITL (Human-in-the-Loop) 交互确认

模型解析机制:

  • ModelRegistry.resolve("openai:gpt-4o") 三级解析：已注册实例 → 工厂缓存 → SPI ServiceLoader
  • 模型供应商模块通过 META-INF/services/io.agentscope.core.model.spi.ModelProvider 注册
  • 模型 ID 格式："provider:model-name"（如 openai:gpt-4o, dashscope:qwen-max）

6.2 agentscope-harness — 增强运行时

内部依赖: agentscope-core (compile)

核心外部依赖:

依赖 用途
org.xerial:sqlite-jdbc 本地工作区索引
org.apache.commons:commons-compress Tar 解压 (沙箱)
redis.clients:jedis (optional) Redis 快照后端
com.aliyun.oss:aliyun-sdk-oss (optional) 阿里云 OSS 快照后端

注意：K8s/Protobuf 等沙箱后端依赖已迁移至 agentscope-extensions-sandbox-* 模块。

包结构 (io.agentscope.harness.agent):

HarnessAgent 中间件栈（按执行序）:

  1. WorkspaceContextMiddleware — 加载 AGENTS.md / MEMORY.md / KNOWLEDGE.md
  2. InboxMiddleware — 待处理消息收件箱
  3. SubagentsMiddleware — 子 Agent 编排
  4. HarnessRuntimeMiddleware — 运行时连接
  5. HarnessSkillMiddleware — 技能加载与策划
  6. AsyncToolMiddleware — 异步工具执行
  7. CompactionMiddleware — 上下文溢出压缩
  8. ToolResultEvictionMiddleware — 旧工具结果淘汰
  9. MemoryFlushMiddleware — 记忆持久化
  10. MemoryMaintenanceMiddleware — 定期记忆维护
  11. SandboxLifecycleMiddleware — 沙箱生命周期
  12. AgentTraceMiddleware — OpenTelemetry 追踪
  13. AtPathExpansionMiddleware — @ 路径展开

8. 架构设计要点总结

  1. Core 是纯粹的核心抽象：Agent 模型、消息系统、工具体系、中间件链、状态管理、权限引擎，零内部依赖，是整个框架唯一的基础

  2. v2 架构三大支柱：

    • Middleware (中间件链) — 替代旧 Hook 体系，提供 onAgent/onReasoning/onActing/onModelCall/onSystemPrompt 五个插入点
    • AgentState (状态管理) — per-session per-user 运行时状态容器，统一管理上下文、权限、工具、任务、Plan Mode 状态
    • Fine-grained Events (细粒度事件) — 34 种 AgentEvent 类型，完整覆盖 Agent 执行生命周期的每个细节

  3. Harness 提供生产级运行时：HarnessAgent 带有 14 个内置中间件的完整运行时栈，工作区管理、技能策划（curator/promoter/gate）、对话压缩、子 Agent 编排等生产级能力

  4. 模型供应商 SPI 插件化：模型供应商从 Core 中完全解耦为独立扩展模块，通过 ModelProvider SPI + ServiceLoader 自动发现，支持 5 种 LLM 供应商 + 任意兼容 OpenAI API 的服务

  5. Extensions 功能域分组管理：~50 个子模块按功能域分为 19 个分组父 POM（model/rag/mem/channel/sandbox/scheduler/nacos/skills/protocol），结构清晰，按需引入

  6. agentscope-all 提供一站式依赖：Shaded uber-JAR 将 core + harness + 全部 extensions 合并为一个 artifact，简化消费者的依赖管理

  7. 多框架集成：Spring Boot 10 个 Starter（含 per-model Starter）、Micronaut 扩展、Quarkus 扩展（runtime + deployment）

  8. 多 LLM 厂商支持：OpenAI（兼容 DeepSeek/GLM）、DashScope（通义千问）、Gemini、Anthropic（Claude）、Ollama 本地模型

  9. A2A 协议深度集成：Google Agent-to-Agent 协议全链路支持（客户端/服务端/RocketMQ 传输/Nacos 服务发现/Spring Boot 自动配置）

  10. 丰富的协作通道：钉钉、飞书、企业微信、GitHub、GitLab 五大通道集成，支持多 Agent 协作场景

  11. 阿里云 + 腾讯云生态：DashScope、百炼（记忆/RAG）、Nacos、OSS、RocketMQ + 腾讯云 COS，覆盖国内两大云厂商
