ubuntu下mysql5.7安装后无法远程登录问题

Posted on 2019-03-23 Edited on 2021-01-31 In mysql Views:

在ubuntu18.04上安装mysql5.7后通过workbench连接和×mysql -u root -p 命令连接均无法连接。命令行显示如下错误：

通过 sudo mysql进行访问，进入命令行查看用户信息：

可以看到此时root用户的认证方式为auth_socket，与其他不一样，原因是在使用ubuntu安装时，mysql默认使用了UNIX auth_socket plugin插件，也就是说着当db_users使用数据库时，将会通过系统用户认证表进行认证。，可参考官方文档

那么如何解决呢？可参考ERROR 1698 (28000): Access denied for user ‘root’@’localhost’,有如下两种方案，设置后，service mysql restart即可。

  • 修改root用户的认证方式
  • 创建一个与系统用户一致的新的数据库用户
